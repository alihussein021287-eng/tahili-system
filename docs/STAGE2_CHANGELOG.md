# Stage 2 Changelog

تغييرات المرحلة 2 ركزت على تحسين سير العمل التشغيلي والواجهات الإدارية بدون migrations إلزامية.

## Reception workflow

- تحسين متابعة الاستقبال والطابور والتنبيهات المرتبطة بالحضور.
- توضيح حالات المراجع وروابط الانتقال إلى الصفحات التفصيلية.

## Appointments

- تحسين عرض المواعيد وحالاتها.
- دعم أوضح لحالات مكتمل، ملغى، لم يحضر، ومجدول ضمن التقارير والتنبيهات.

## Tasks/notifications

- تحسين صفحة المهام بالفلاتر حسب المسؤولية، الدور، الحالة، التأخير، والإنجاز.
- تحسين إنشاء المهمة بإسناد لمستخدم أو دور، أولوية، تاريخ استحقاق، وربط اختياري بمراجع.
- دعم إجراءات بدء، إنجاز، إعادة فتح، وحذف للأدمن.
- ربط إنشاء المهمة بالإشعارات عند توفر جدول notifications بدون migration.

## Devices maintenance

- تحسين صفحة الأجهزة بفلاتر الصيانة، التسليم، الاستبدال، والمراجع.
- إضافة إجراءات تسجيل الصيانة، تعليم الجهاز كمستبدل، وحذف للأدمن.
- إبراز الأجهزة التي تحتاج صيانة وربطها بتنبيهات الجاهزية ولوحة النظام.
- دعم إنشاء مهمة صيانة عند توفر صلاحية tasks.create.

## Daily reports

- تحسين التقرير اليومي للإدارة ليعرض زيارات، مواعيد، طابور، مهام، صيدلية، مخزون، أجهزة، ورقود اليوم.
- إضافة فلتر التاريخ وتحسين الطباعة برأس رسمي وملخص أرقام وجداول مختصرة.

## Sick leave approvals

- تحسين عرض الإجازات المرضية داخل ملف المراجع.
- توضيح حالة اعتماد اللجنة والمدير وإظهار من اعتمد ومتى.
- تحسين الطباعة الرسمية بصياغة RTL ورأس رسمي وبيانات المراجع والتشخيص والمدة والتواقيع.

## Users/permissions UI

- تحسين صفحة المستخدمين بالبحث والفلترة وعرض الحالة والدور وآخر دخول.
- تحسين صفحة المستخدم بملخص الحساب والنشاط والصلاحيات الفعلية والاستثناءات.
- تحسين صفحة الصلاحيات بالمجموعات وقوالب الأدوار والتحذير قبل الصلاحيات الحساسة.
- حصر إدارة المستخدمين والصلاحيات على ADMIN.

## Backup/readiness

- تحسين صفحة النسخ الاحتياطي لعرض آخر نسخة، الحجم، النوع، وحالة النسخ التلقائي.
- إضافة تأكيد نصي للاستعادة والحذف وتحذير واضح قبل الاستعادة.
- توضيح الفرق بين نسخ قاعدة البيانات من الواجهة ونسخ السيرفر التي تشمل uploads.
- تحسين readiness لإظهار آخر backup والتحذير عند قدم النسخة أو توقف النسخ التلقائي.
- إضافة توثيق تشغيل backup وcron والاستعادة الآمنة في `docs/BACKUP_OPERATIONS.md`.
